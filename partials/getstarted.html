Introduction

The following instructions will allow to define a REST API and deploy it with erocci without any code.

Download on Github

Requirements

  • erlang (>=18)
  • rebar
  • OpenSSL lib + headers
  • libxml2 lib + headers
  • libexpat lib + headers

On a Debian box, the following will do the trick:

wget http://packages.erlang-solutions.com/erlang-solutions_1.0_all.deb && sudo dpkg -i erlang-solutions_1.0_all.deb sudo apt-get update sudo apt-get install build-essential erlang libxml2-dev

Building

make

Simple, isn't it ?

Setup

The default config file is located in erocci/config/sys.config.

For a basic configuration, just replace the schema path to point to your XML file.

You can read from an absolute path : 

			{schema, "/absolute/path/to.xml"}

Or from a distant repository : 

			{schema, {extension, "http://example.org/occi/schema#"}}

Run

$ ./start.sh  --help
Usage: ./start.sh [-d] [-t] [-s] [-x <jid>] [-c <config>] [-h]
	-d           Print debug messages
	-q           Print only error messages
	-t           Start HTTP listener
	-s           Start HTTPS listener (default: HTTP)
	-x <jid>     Start XMPP listener with given JID
	-c <config>  Set alternate config file (default: example.config)
	-h           Print this help
